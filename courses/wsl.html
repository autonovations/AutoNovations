¿Qué es WSL?

Nivel 1 · Principiante

El Windows Subsystem for Linux (WSL) es una capa de compatibilidad desarrollada por Microsoft que permite ejecutar un entorno GNU/Linux directamente sobre Windows, sin necesidad de máquinas virtuales ni dual boot.

Esto significa que puedes usar herramientas como bash, grep, git, python y cualquier aplicación Linux en tu máquina Windows sin salir de tu entorno habitual.

WSL vs WSL 2
WSL 1 traduce llamadas de sistema Linux a llamadas de Windows. WSL 2 usa un kernel Linux real dentro de una máquina virtual ligera (Hyper-V), siendo mucho más rápido y compatible. En 2024+, siempre usa WSL 2.

Arquitectura de WSL 2

WSL 2 corre un kernel Linux completo en una VM Hyper-V optimizada y liviana. Windows y Linux comparten el mismo sistema de archivos mediante un servidor 9P (protocolo de red para archivos).

Rendimiento Nativo
Kernel Linux real. Compilaciones hasta 20x más rápidas que WSL 1.
Compatibilidad Completa
Soporta syscalls de Linux al 100%, incluyendo Docker, ptrace y más.
Integración Windows
Accede a archivos de Windows desde Linux y viceversa sin fricción.
GUI Linux (WSLg)
Ejecuta aplicaciones gráficas Linux directamente en tu escritorio Windows.

¿Para quién es WSL? Para desarrolladores, ingenieros de datos, estudiantes de DevOps, o cualquier persona que necesite herramientas Linux pero prefiera usar Windows como sistema principal. Es ideal para aprender Linux sin arriesgar tu sistema.
Quiz — Módulo 1
1. ¿Qué versión de WSL se recomienda usar actualmente y por qué?
WSL 2 corre un kernel Linux completo en una VM Hyper-V ligera, lo que le da compatibilidad total con syscalls de Linux y un rendimiento muy superior para compilaciones y E/S en el sistema de archivos de Linux.
2. ¿Cómo comparte archivos WSL 2 entre Windows y Linux?
WSL 2 usa el protocolo de red 9P (Plan 9 Filesystem Protocol) para montar el sistema de archivos de Windows en Linux (/mnt/c, /mnt/d) y viceversa, permitiendo acceso bidireccional en tiempo real.

Instalación de WSL 2

Nivel 1 · Principiante
Requisitos previos: Windows 10 (build 19041+) o Windows 11. Virtualización habilitada en BIOS/UEFI. Privilegios de administrador.

Método 1: Un Solo Comando (Recomendado)

Desde Windows 10 build 2004 en adelante, puedes instalar WSL con un único comando en PowerShell como Administrador:

powershell (administrador) 
# Instala WSL 2 con Ubuntu por defecto
wsl --install

# Si quieres elegir la distribución:
wsl --install -d Ubuntu-22.04
1
Abre PowerShell como Administrador
Presiona Win+X y selecciona "Terminal de Windows (Administrador)" o "Windows PowerShell (Administrador)".
2
Ejecuta el comando de instalación
Copia y pega wsl --install. Windows descargará e instalará WSL 2 y Ubuntu automáticamente.
3
Reinicia el sistema
Cuando termine la instalación, reinicia Windows. WSL requiere un reinicio para completar la configuración del kernel.
4
Configura tu usuario Linux
Ubuntu se abrirá automáticamente y te pedirá crear un nombre de usuario y contraseña para tu entorno Linux.

Sistema de Archivos en WSL

Nivel 2 · Intermedio

Uno de los aspectos más importantes de WSL es entender cómo se integran los sistemas de archivos de Windows y Linux. Ambos son accesibles desde cualquiera de los dos entornos.

Jerarquía del Sistema de Archivos Linux

/ # Raíz del sistema ├── home/ # Directorios de usuarios │ └── usuario/ # Tu home (~) ├── mnt/ # Montajes — aquí están tus discos Windows │ ├── c/ # Disco C: de Windows │ └── d/ # Disco D: de Windows ├── etc/ # Configuración del sistema ├── var/ # Datos variables (logs, cache) ├── usr/ # Aplicaciones y librerías ├── tmp/ # Archivos temporales ├── root/ # Home del superusuario └── proc/ # Sistema de archivos virtual (procesos)

Acceso Cruzado entre Windows y Linux

bash — Acceso a Windows desde Linux 
# Acceder al escritorio de Windows
cd /mnt/c/Users/TuNombre/Desktop

# Acceder a Documentos
cd /mnt/c/Users/TuNombre/Documents

# Listar archivos de C:
ls /mnt/c/

# Copiar archivo de Windows a Linux
cp /mnt/c/Users/TuNombre/archivo.txt ~/

# Crear alias para acceso rápido
echo 'alias cdesktop="cd /mnt/c/Users/TuNombre/Desktop"' >> ~/.bashrc
source ~/.bashrc
powershell / explorer — Acceso a Linux desde Windows 
# Abrir el home de Linux en el Explorador de Archivos
explorer.exe .     # Desde la terminal WSL

# En PowerShell, acceder a Linux via UNC path
dir \\wsl$\Ubuntu-22.04\home\usuario

# En el Explorador: busca "Linux" en el panel izquierdo
# O escribe en la barra: \\wsl$\Ubuntu-22.04
Regla de Oro de Rendimiento: Guarda tus proyectos dentro del sistema de archivos Linux (~/) y no en /mnt/c/. Las operaciones de I/O dentro de Linux son hasta 10x más rápidas que en la ruta montada de Windows.

Permisos de Archivos en Linux

Linux usa un sistema de permisos con tres categorías: propietario, grupo y otros. Cada uno puede tener permisos de read, write y execute.

bash — Permisos 
# Ver permisos
ls -la script.sh
-rwxr-xr-- 1 usuario grupo 1234 Jun 26 09:00 script.sh
# │││├─ usuario: rwx (leer, escribir, ejecutar)
# ││├── grupo: r-x (leer, ejecutar)
# │└─── otros: r-- (solo leer)

# Cambiar permisos (chmod)
chmod +x script.sh      # Hacer ejecutable
chmod 755 script.sh     # rwxr-xr-x (octal)
chmod 644 archivo.txt   # rw-r--r--
chmod -R 755 directorio/ # Recursivo

# Cambiar propietario (chown)
sudo chown usuario:grupo archivo.txt
sudo chown -R usuario directorio/
Quiz — Módulo 4
1. ¿En qué ruta está montado el disco C: de Windows dentro de WSL?
WSL monta automáticamente todas las unidades de Windows en /mnt/. El disco C: está en /mnt/c, el D: en /mnt/d, etc.
2. ¿Qué hace chmod 755 script.sh?
755 en octal = 7(rwx) para propietario, 5(r-x) para grupo, 5(r-x) para otros. Es el permiso estándar para scripts y directorios públicos.
✅ ¡Módulo 4 completado! Dominas el sistema de archivos.

Configuración Avanzada de WSL

Nivel 2 · Intermedio

WSL es altamente configurable. Puedes ajustar recursos de hardware, comportamiento del sistema de archivos, integración con Windows y mucho más mediante archivos de configuración.

Archivo .wslconfig (Global)

Este archivo en C:\Users\TuNombre\.wslconfig controla los recursos globales de todas las distribuciones WSL.

ini — C:\Users\TuNombre\.wslconfig 
[wsl2]
# Memoria RAM asignada a WSL (por defecto: 50% del sistema)
memory=8GB

# Núcleos de CPU (por defecto: todos los del sistema)
processors=4

# Memoria de swap
swap=2GB

# Habilitar soporte para GUI apps (WSLg)
guiApplications=true

# Networking — mirrored para mejor integración
networkingMode=mirrored

# Auto-reclamar memoria no usada
autoMemoryReclaim=gradual

# Habilitar DNS tunneling
dnsTunneling=true

# Habilitar firewall de Windows
firewall=true

Archivo /etc/wsl.conf (Por Distribución)

Este archivo dentro de cada distribución Linux controla su comportamiento individual.

ini — /etc/wsl.conf (dentro de WSL) 
[automount]
# Montar drives de Windows automáticamente
enabled = true
root = /mnt/
# Opciones de montaje (drs: acceso a metadatos Windows)
options = "metadata,umask=22,fmask=11"
mountFsTab = true

[boot]
# Ejecutar systemd (WSL2 con kernel 5.15+)
systemd = true
# Ejecutar script al iniciar
command = "service ssh start"

[network]
# Nombre del host en la red
hostname = mi-wsl-dev
# Generar /etc/hosts automáticamente
generateHosts = true
generateResolvConf = true

[interop]
# Permitir ejecutar binarios de Windows desde Linux
enabled = true
appendWindowsPath = true

Personalizar el Shell con ~/.bashrc

bash — ~/.bashrc 
# Agregar al final de ~/.bashrc

# Alias útiles

Quiz — Módulo 6
1. ¿Dónde se guarda el archivo .wslconfig global?
.wslconfig es un archivo de Windows que va en tu carpeta de usuario (%USERPROFILE%). Controla los recursos globales de WSL 2 como memoria y CPU.

En la sección [boot] de /etc/wsl.conf, la opción systemd = true activa el gestor de servicios systemd, disponible desde el kernel de WSL 5.15+.

Redes y Puertos en WSL

Nivel 2 · Intermedio

WSL 2 tiene su propia interfaz de red virtual. Entender cómo funcionan las redes te permite exponer servicios, conectarte a bases de datos y configurar proxies correctamente.

Conceptos de Red en WSL

localhost
Un servicio en WSL es accesible desde Windows en localhost:puerto. WSL redirige automáticamente.
IP de WSL
WSL tiene su propia IP interna (ej: 172.x.x.x). Cambia con cada reinicio en modo NAT.
Modo Mirrored
Con networkingMode=mirrored, WSL comparte la IP de Windows, simplificando enormemente la red.
bash — Comandos de Red 
# Ver IP de WSL
ip addr show eth0
hostname -I
172.20.48.1

# Ver puertos en uso
ss -tulpn
netstat -tulpn

# Probar conectividad
ping google.com -c 4
curl -I https://google.com

# Desde WSL, obtener IP del host Windows
cat /etc/resolv.conf | grep nameserver
nameserver 172.20.48.1   # Esta es la IP del host Windows

# Configurar una variable de entorno con la IP del host
export HOST_IP=$(cat /etc/resolv.conf | grep nameserver | awk '{print $2}')
echo $HOST_IP

Quiz — Módulo 7
1. Si levantas un servidor en WSL en el puerto 3000, ¿cómo accedes desde Windows?
WSL 2 redirige automáticamente los puertos de Linux al host Windows. Cualquier servicio que escuche en 0.0.0.0 o localhost en WSL es accesible desde Windows en localhost:puerto.

Script Real: Setup de Proyecto Dev

bash — setup_project.sh 
#!/bin/bash
# Script para configurar un nuevo proyecto Python

set -e  # Salir si hay algún error

PROJECT_NAME=${1:-"mi_proyecto"}  # Parámetro o default
PYTHON=python3

echo "🚀 Creando proyecto: $PROJECT_NAME"

# Crear estructura
mkdir -p $PROJECT_NAME/{src,tests,docs}
cd $PROJECT_NAME

# Entorno virtual
$PYTHON -m venv .venv
source .venv/bin/activate

# Archivos base
cat > README.md <<EOF
# $PROJECT_NAME
Proyecto creado con WSL + AutoNovations setup script.
EOF

touch src/__init__.py tests/__init__.py
echo "requests\npytest" > requirements.txt
pip install -r requirements.txt -q

# Inicializar git
git init && git add . && git commit -m "Initial commit"

echo "✅ Proyecto $PROJECT_NAME listo en $(pwd)"

Quiz — Módulo 9
1. ¿Qué hace la línea #!/bin/bash al inicio de un script?
El shebang (#!) seguido de la ruta del intérprete dice al kernel qué programa usar para ejecutar el script. #!/bin/bash usa Bash, #!/usr/bin/env python3 usaría Python.
2. ¿Cómo capturas la salida de un comando en una variable Bash?
La sintaxis $(...) se llama "command substitution". Ejecuta el comando dentro y captura su salida stdout como string. Por ejemplo: FECHA=$(date)

Exportar e Importar Distribuciones

Puedes crear backups completos de tu distribución WSL y restaurarlos en otro equipo.

powershell — Backup y Restore 
# Exportar distro a archivo .tar
wsl --export Ubuntu-22.04 C:\backups\ubuntu-dev.tar

# Importar en otro equipo o con otro nombre
wsl --import Ubuntu-Dev C:\WSL\Ubuntu-Dev C:\backups\ubuntu-dev.tar

# Verificar importación
wsl --list --verbose

# Iniciar la distro importada
wsl -d Ubuntu-Dev

# Instalar múltiples distribuciones

wsl --install -d kali-linux

wsl --terminate Debian

# Eliminar una distro completamente
wsl --unregister Debian

Quiz Final — Módulo 10
1. ¿Qué comando exporta una distribución WSL completa para hacer un backup?
wsl --export <Distro> <archivo.tar> guarda el sistema de archivos completo en un tar. Luego puedes restaurarlo en cualquier equipo con wsl --import.
2. ¿Qué tipo de clave SSH se recomienda usar hoy en día para mayor seguridad?
Ed25519 usa curvas elípticas y es más seguro y rápido que RSA. Es el estándar recomendado por GitHub, GitLab y la mayoría de servicios modernos. Genera claves más cortas pero con mayor seguridad criptográfica.
